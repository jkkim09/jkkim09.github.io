처음 문제를 보고 의외로 간단할거 같아 코드를 치다 중간쯤 되었을때 처음 생각하지 못했던 예외사항 들이 생겨서 60% 작성한 코드를 접고 다시 지우고 노트에 예외사항들을 적어가며 코드를 다시 작성하였다.
내가 생각하지 못한부분은 text length 길이만큼 반복하는것이아니라 text length 의 절반만큼만 반복하면 된다는 것이였다 예를들어 14자리 text 면 1~7까지만 그룹핑 해서 계산하면 되는것이였다 그리고 생가하지 못한부분은
단순 text를 더하는 것이 아니라 앞단과 뒷단의 text 를 더할때 조건이 필요하다는 것이였다. 풀이 CODE는 위 링크에 마지막 버전을 보면 된다.(quiz1-3.js). 물론 코드는 마음에 들지안으며 분명 개선사항은 있을 것이다. 다른 사람의 코드를 보지 안아 어느정도의 코드인지는 모르겠다.
2. KaKao 2020 문제 정답률: 23.1%
정답코드 https://github.com/jkkim09/quiz-javascript/tree/master/quiz-2
위 문제를 처음 보았을때 처음 생각한것은 가로 ')' , '(' 의 문자의 순서를 맞추면 되겠구나 생각을했다. 그러나 예문의 u, v 값을 어떡게 불리하는지 감이오지 안았다 그래서 가로 문자를 문자로 보는것이아니라
'(' 를 +1 로 ')' 를 -1 로 보았을때 0 이 될때가 짝이맞는 다는 공식을 발견하게 되었다. 그리고 그 짝으로로 시작되는 시작 index 와 끝나는 index 값을 알아내고 시작 index 의 문자가 ')' 사직 되면 시작 - 끝 의 길이의 반만큼의 문자열을
앞뒤를 바꾸어주면 올바른 괄호 문자열이 되는것을 확인하였다. quiz2-1.js 는 예문에서 처럼 재귀적으로 처리하지안았다. 그래서 완벽한 답은 아닌것 같아 quiz2-2.js를 만들어 다시 풀어보려고 한다. 일단 내가 푼 방식은 위에 링크에서 볼 수 있다.
