html modal
일반적으로 web에서 modal 을 뛰울때는 alert('test') 을 이용하여 원하는 문구를 뛰운다. 하지만 화면 개발을 하다보면 디자이너가 특정 디자인으로 메세지를 보여주고 싶어할때가 대부분이다.
그러기 때문에 web에서는 html을 이용하여 modal 을 만들어 보여준다. 직접 물론 많은 사람들이 만들어 공유하고 있는 modal 들이 있지만 불필요한 기능 이나 코드가 들어가고 확장성을 보장 할 수 없다고 생각한다. 그래서 저는 직접 modal 기능을 만들어 사용합니다.
위 modal util 은 메세지나 버튼의 문구를 커스텀 할 수 있고 버튼 클릭스 callback 을 통해서 다음 로직을 진행 할 수 있게 만들었습니다.
차 후 version 은 어느정도 화면 모양도 커스텀 가능하게 만들거나 버튼 수를 변경 할 수있게등 추가할 생각입니다.
mouse & touch scroll event
위 scroll util 은 자식 element 가 부모 element 범위를 넘어가면 화면이 잘리는 영역이 생기게 된다. 그러할경우 mouse 혹은 mobile touch 이벤트를 통해서 상하(아직 적용못하였다. ㅠㅠ 추 후 진행) 좌우 로 자식 element 의 position을 이동할 수 있다.
어느정도 정확성 있게 이벤트가 작동하는것을 볼 수 있고 추가적으로 개발할 부분은 이동 스피드에따라 부드럽게 이동 할 수있게 하는기능과 이동 거리만 반환하는 callback 이벤트를 기능이 필요 할 것으로 보인다.
사용법은 위의 my-util링크로 들어가 확인 할 수 있다.
web javascript validation
WEB서비스를 만들다보면 가입에필요한 이름 생년월일 결제에필요한 카드정보 등 여로정보를 입력받을 일이많다. 단순하게 입력받은 값을 back-end 단으로 넘겨주는 사람이 많은데 난 최소한의 개발자라면 front-end 단에서 값에 대한 validation 을 진행 하고 값을 보내야한다고 생각한다.
물론 back-end 단에서도 들어온 값에대한 기본적인 validation 을 해야한다. 그래서 항상 서비스를 만들면서 그떄그때 validation 을 만들어서 진행하다. 공통적으로 쓸 수 있는 validation util을 만들었는데 항상 github 에 올려 두며 나중에 쓰려고 생각했지 실제로 올려두지 못해 다시 코드를 작성하게 되었다.
아직은 구조를 바꾸어서 개발중이고 수일 내로 만들어서 올릴예정이며 사용법도 정리 할 예정이다. 구조를 더 좋게 만들려니 어디까지 생각해서 만들어야할지 고려중이다.
