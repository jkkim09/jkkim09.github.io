나의 web front-end 정리

WEB Front-End 기본 기술은 html, css, javascript, jquery(?할 줄 알지만 별로 쓰고싶은 생각은없다.) 정도만 할 수 있어도 화면을 동작 시키거나
통신을 하거나 하는 서비스는 만들 수 있다. 하지만 계속 개발을 하면서 힘든 부분들이 생길 것이다. 화면조작도 직접 해야하고 back-end 에서 받은 값또한 직접 조작해야하는
작업들이 많기 때문이다. 그래서 요즘은 MVVM(Model + View + View Model) 이라는 계념이 나오면서 화면 조작과 data 조작을 분리 시켜 모듈 적으로 만들 수 있게 되었다.

내가 jquery를 안쓰는 이유

2017.03.06일 개발자를 시작했을떄 WEB Front-End 로 개발자 인생을 시작하였다. 그 당시 너무 초짜라 할줄 아는게 별로없어 간단하게 개발 가능한 Jquery 위주로 많이 사용했었다.
처음에는 성능이며 차 후 유지보수 등 중요한 것들이 신경쓰지 못하여 일단 돌아가게 만드는것이 고작이였지만 점차 성능, 유지보수, 가독성 등을 생각하니 Jquery를 쓰는것이 안좋다는 생각을했다.
일단. $('.class') $(#id) 등 처음 보는 사람은 무엇을 의미하는지 알 수 가 없었다 그이유는 언어적으로 명시되어있는게 아니라 형식으로 명시하기때문이다.
반면 javascript는 document.getElementById, document.getelementsbyclassname 등 언어적으로 알수있는 방식으로 되어있다. id 로 하여금 element 가져오는구나, class 로 하여금 element 가져오는구나.
이런식으로 딱봐도 알수 있는 형식으로 되어있다. 물론 Jquery보다는 코드가 길다. 하지만 난 단점이라고 생각하지 않는다.
이외에 성능적으로 봐도 Jquery로 처리하는것보다 javascript로 처리하는것이 더 빠르기도하며 (DOM에 접근할때 등) 자바스크립트로만 짜는데 이점들이 더많다.

2018 프론트엔드 로드맵에 jQuery가 제외된걸봐도 Jquery의 입장을 알 수 있다.
